# Contributing

感谢你关注这个项目。

这个仓库目前是一个偏实战导向的 ACM 榜单后端服务，欢迎通过 Issue、Discussion 或 Pull Request 参与改进。为了让协作更顺畅，建议在提交前先阅读下面的约定。

## 你可以贡献什么

- 修复 bug
- 改进 PTA / Kafka 相关稳定性
- 补充测试、文档和部署脚本
- 优化接口设计、日志和可维护性
- 改善前后端协作体验

## 提交前建议

1. 先开一个 Issue，简单说明你想解决的问题。
2. 如果改动较大，先对方案达成一致再开始实现。
3. 保持提交范围聚焦，尽量一个 PR 只解决一类问题。

## 本地开发

### 1. 安装依赖

```bash
pip install -e .
```

### 2. 配置环境变量

```bash
cp .env.example .env
```

请不要把真实 `.env` 提交到仓库。

### 3. 启动服务

```bash
uvicorn app:app --host 0.0.0.0 --port 8090
```

## 代码风格

- 保持改动尽量小而明确
- 优先写可读性高的代码
- 不要把真实 PTA 凭据、Kafka 地址密钥或隐私数据提交到仓库
- 新增配置优先放入 `.env.example`
- 修改接口时，请同步更新 README

## Pull Request 说明

提交 PR 时，建议包含以下信息：

- 改动背景
- 主要修改内容
- 是否涉及配置变更
- 是否影响现有接口或 Kafka 消息格式
- 测试方式

## 安全相关

如果你发现的是安全问题或凭据泄露风险，建议不要直接公开提 Issue，可以先私下联系维护者处理。

## 沟通原则

- 对问题本身保持具体
- 对协作者保持友善
- 对历史代码保持尊重

感谢你的贡献。
